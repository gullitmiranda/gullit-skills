# Workflow Details

## Feature From Idea

Use when the user starts with an ambiguous feature, product problem, or domain
change.

Expected result:

- Domain terms clarified.
- Plan or PRD created.
- Work split into vertical slices.
- Runtime selected before implementation.
- Slice implemented and validated.

## Bug Diagnosis

Use when the user reports a bug, regression, exception, failing test, or
performance issue.

Expected result:

- Reproducible feedback loop.
- Proven cause.
- Fix.
- Regression test when a correct test surface exists.
- Validation evidence.

## Existing Plan To PR

Use when there is already a clear plan, issue, or PRD.

Expected result:

- Source artifact followed without unnecessary re-planning.
- Work isolated when needed.
- Implementation validated.
- PR prepared when requested.

## Parallel Workstreams

Use when multiple plans, issues, or investigations are active in the same repo
or workspace.

Expected result:

- Workstream list.
- Runtime owner per workstream.
- Context capsule for each child agent/tool.
- Distilled results returned to the parent chat.

## Architecture Improvement

Use when the code is hard to test, change, understand, or delegate.

Expected result:

- Architecture candidates.
- Durable decisions captured only when needed.
- Refactor split into reviewable increments.
- Validation through improved interfaces.
